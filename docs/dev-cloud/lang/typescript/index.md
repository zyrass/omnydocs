## Présentation

!!! quote "Analogie pédagogique — Le Plan d'Architecte"
    JavaScript est comme construire à vue : vous posez les briques en espérant que la structure tiendra. TypeScript est le plan d'architecte : chaque pièce du bâtiment (variable, fonction, objet) est documentée, ses dimensions (types) sont vérifiées avant que la première brique soit posée. Si vous essayez de poser une porte là où doit aller une fenêtre, TypeScript le détecte **avant la construction** (avant le runtime).

**TypeScript** est JavaScript avec un système de types statiques. Il se **transpile** en JavaScript et n'existe qu'à l'étape de développement — à runtime, c'est du JS standard.

| Caractéristique | Valeur |
|---|---|
| **Créé par** | Anders Hejlsberg, Microsoft (2012) |
| **Philosophie** | Types optionnels, superensemble de JS |
| **Transpileur** | `tsc` (TypeScript Compiler) |
| **Cas d'usage** | Tout projet JS > 500 lignes, équipes, APIs |
