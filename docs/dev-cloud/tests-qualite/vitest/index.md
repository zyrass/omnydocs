## Introduction

!!! quote "Pourquoi Vitest ?"
    Vitest est le framework de tests unitaires natif de l'écosystème Vite. Compatible avec la syntaxe Jest (migration facile), ultra-rapide grâce au HMR Vite, et TypeScript-first. Si vous utilisez Vite pour votre frontend (Vue, React, Alpine.js + Vite), Vitest est le choix naturel.

Vitest se positionne comme le successeur de Jest dans l'écosystème Vite/moderne. Il partage la même API (`describe`, `it`, `expect`) mais offre des performances bien supérieures grâce à l'intégration native avec Vite.

## Pré-requis

- JavaScript ES6+ (fonctions, modules, async/await)
- Notions de base sur les tests ([TDD →](../tdd.md))
- Un projet Vite existant (Vue, React, ou vanilla)

## Ce que vous saurez faire

- Configurer Vitest dans un projet Vite existant
- Écrire des tests unitaires pour des fonctions, modules et composants
- Mocker des dépendances (API, modules, timers)
- Générer des rapports de coverage et les intégrer en CI/CD
