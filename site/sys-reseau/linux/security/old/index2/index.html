Sécurité (Host)¶

Introduction¶

Analogie pédagogique

Un serveur Linux, c'est une forteresse médiévale multicouches. Les remparts extérieurs (pare-feu) filtrent les attaquants. Les gardes aux portes (fail2ban) bannissent les assaillants répétés. L'inspection des fondations (Lynis) vérifie que la construction est solide. Le contrôle des provisions (Vuls) détecte les vivres contaminées (packages vulnérables). Les vigiles intérieurs (ClamAV/LMD) cherchent les infiltrés. Les détectives (chkrootkit) traquent les passages secrets (rootkits). Et les chroniqueurs (logs) documentent tout. Sans toutes ces couches, une seule faille suffit à tout compromettre.

La sécurité hôte Linux regroupe toutes les mesures qui protègent directement la machine : configuration système, réseau local, permissions, services, vulnérabilités packages, détection malware, mises à jour, et outils de surveillance. L'objectif n'est pas d'ajouter "un antivirus" et de croire que c'est réglé, mais de construire une défense en profondeur : prévention, réduction de surface d'attaque, détection vulnérabilités, détection malware, et réaction.

Objectifs du parcours¶

À la fin de cette section, tu seras capable de comprendre et mettre en place :

  • Périmètre réseau : pare-feu cohérent et maintenable (UFW/nftables)
  • Protection bruteforce : bannissement automatique attaques (fail2ban)
  • Audit configuration : durcissement systématique avec scoring (Lynis)
  • Scan vulnérabilités : détection CVE packages installés (Vuls)
  • Détection malware général : antivirus multi-usage (ClamAV)
  • Détection malware web : webshells et backdoors serveurs (LMD)
  • Détection rootkits : compromission système profonde (chkrootkit)
  • Supervision : logs exploitables et alerting (journald/rsyslog)
  • Routine opérationnelle : checklist hebdo/mensuelle automatisée

Vue d'ensemble des chapitres¶

Phase 1 : Prévention et Périmètre¶

  • 01 — Surface d'attaque & baseline

    Inventaire des services, ports, utilisateurs, et règles minimales de sécurité. Objectif : savoir "ce qui tourne" avant de durcir.

    Niveau : 🟢 | Durée : 30-45 min

    Accéder

  • 02 — Pare-feu (UFW / nftables)

    Stratégie de filtrage, whitelisting, règles persistantes, erreurs classiques. UFW pour démarrer, nftables pour contrôler finement.

    Niveau : 🟢→🟡 | Durée : 60-90 min

    Accéder

  • 03 — Anti-bruteforce (fail2ban)

    Protection contre brute force SSH/web/mail, scans agressifs, abus applicatifs. Jails, filtres, bannissement automatique, bonnes pratiques et limites.

    Niveau : 🟡 | Durée : 60-90 min

    Accéder

Phase 2 : Audit et Détection Vulnérabilités¶

  • 04 — Audit configuration (Lynis)

    Audit complet : scoring hardening, recommandations sécurité, baseline système, export rapports, intégration CI. Comprendre ce que Lynis dit et ce qu'il ne dit pas.

    Niveau : 🟡→🔴 | Durée : 90-120 min

    Accéder

  • 05 — Scan vulnérabilités (Vuls) ⭐ NOUVEAU

    Scanner CVE packages système : détection 200,000+ vulnérabilités connues, scoring CVSS, alertes critiques, patch management workflow, compliance PCI-DSS.

    Niveau : 🟡→🔴 | Durée : 90-120 min

    Accéder

Phase 3 : Détection Malware (Général → Spécialisé → Profond)¶

  • 06 — Antivirus général (ClamAV)

    Comprendre l'intérêt réel sur Linux, cas d'usage (partage fichiers, mail gateway, scans serveurs), planification, performance, faux positifs. 8M+ signatures.

    Niveau : 🟢→🟡 | Durée : 60-90 min

    Accéder

  • :lucide-spider:{ .lg .middle } 07 — Malware web (LMD) ⭐ NOUVEAU

    Détection webshells (c99, r57, WSO), backdoors PHP/Python, cryptominers, SEO spam. 17,000+ signatures serveurs web, monitoring temps réel uploads, intégration cPanel/Plesk.

    Niveau : 🟡→🔴 | Durée : 90-120 min

    Accéder

  • 08 — Rootkits système (chkrootkit)

    Détection rootkits kernel-level, binaires trojaned, processus cachés. Limites, interprétation résultats, stratégie réaliste. Comment éviter de tomber dans le piège "tout est compromis".

    Niveau : 🟡 | Durée : 45-60 min

    Accéder

Phase 4 : Supervision et Opérations¶

  • 09 — Logs & supervision

    Journaux système, journalctl, rotation, points de contrôle, alerting. Préparer le terrain pour Wazuh / SIEM. Logs exploitables et corrélation événements.

    Niveau : 🟡→🔴 | Durée : 60-90 min

    Accéder

  • 10 — Synthèse : routine sécurité 🎯

    Checklist hebdo/mensuelle automatisée, procédures incident, durcissement progressif, métriques. Objectif : rendre la sécurité "opérationnelle", pas théorique.

    Niveau : 🔵 Synthèse | Durée : 60-90 min

    Accéder

Principes structurants¶

  1. Inventaire d'abord : on ne durcit pas à l'aveugle, d'abord baseline
  2. Périmètre ensuite : pare-feu + anti-bruteforce (accès contrôlés)
  3. Audit configuration : durcissement mesurable (Lynis scoring)
  4. Scan vulnérabilités : détection CVE packages (Vuls)
  5. Détection malware progressive :
  6. ClamAV = malware général (8M+ signatures tous types)
  7. LMD = malware web spécialisé (17K+ webshells/backdoors)
  8. chkrootkit = rootkits système profonds (kernel-level)
  9. Supervision continue : logs exploitables, alertes, métriques
  10. Cycle répété : la sécurité hôte est un cycle, pas une action unique

Pourquoi cet ordre ?

  1. Bash = Fondamental pour comprendre et créer scripts automation
  2. Baseline = Inventaire (impossible de sécuriser l'inconnu)
  3. UFW = Périmètre réseau (première ligne défense)
  4. fail2ban = Protection active bruteforce
  5. Lynis = Audit configuration (hardening systématique)
  6. Vuls = Scan CVE packages (vulnérabilités connues)
  7. ClamAV = Antivirus général (baseline détection malware)
  8. LMD = Malware web spécialisé (serveurs web focus)
  9. chkrootkit = Rootkits profonds (compromission système)
  10. Logs = Supervision (tout consolider)
  11. Synthèse = Routine opérationnelle (automatiser)

Prérequis et environnement¶

Cette section suppose :

  • Linux : Debian/Ubuntu (UFW facile) ou RHEL/Fedora (nftables natif)
  • Accès : sudo/root
  • Services : Au minimum SSH, idéalement web (nginx/apache) pour tester LMD
  • Compréhension : Permissions, systemd, logs, bash basique
  • Temps : 12-16h pour parcours complet (peut être réparti sur 2 semaines)

Approche recommandée

Teste tout dans une VM dédiée (Proxmox, VirtualBox, WSL2). C'est la façon la plus rapide d'apprendre sans risquer un système de production. Clone la VM entre chaque chapitre pour avoir des snapshots "états propres".

# -----------------------------------------------------------------------
# Sécurité hôte Linux - Stack complète 2024
# Outils de durcissement, scan vulnérabilités et détection intrusion
# -----------------------------------------------------------------------

# Périmètre et accès
ufw                 # Uncomplicated Firewall (frontend iptables/nftables)
fail2ban            # Protection bruteforce SSH/web/mail

# Audit et hardening
lynis               # Audit sécurité système (scoring hardening)
debsums             # Vérification checksums packages Debian (complément Lynis)
aide                # File integrity monitoring (complément détection)

# Scan vulnérabilités
vuls                # Scanner CVE packages (200K+ vulnérabilités NVD/OVAL)
go-cve-dictionary   # Database CVE pour Vuls
goval-dictionary    # OVAL database pour Vuls
gost                # Security Tracker pour Vuls

# Détection malware
clamav              # Antivirus open-source (8M+ signatures général)
clamav-daemon       # Daemon ClamAV (clamd)
maldet              # Linux Malware Detect (17K+ webshells/backdoors)
chkrootkit          # Détection rootkits (70+ signatures kernel-level)

# Monitoring et logs
rsyslog             # Centralisation logs
logrotate           # Rotation automatique logs
auditd              # Audit système kernel-level (optionnel)

# Utilitaires
jq                  # Parser JSON (résultats Vuls)
curl                # HTTP client (updates, webhooks)
wget                # Download files
git                 # Version control (scripts automation)

Installation rapide stack complète¶

#!/bin/bash
# install-security-stack.sh
# Installation complète stack sécurité host Linux

set -euo pipefail

echo "=== Installation Stack Sécurité Host Linux ==="

# Update system
sudo apt update

# Périmètre
sudo apt install -y ufw fail2ban

# Audit
sudo apt install -y lynis debsums aide

# Vulnérabilités (Vuls - voir guide dédié pour installation complète)
# Vuls nécessite installation manuelle binaires Go

# Malware
sudo apt install -y clamav clamav-daemon
cd /tmp
wget http://www.rfxn.com/downloads/maldetect-current.tar.gz
tar -xzf maldetect-current.tar.gz
cd maldetect-*
sudo ./install.sh
cd /tmp
wget ftp://ftp.chkrootkit.org/pub/seg/pac/chkrootkit.tar.gz
tar -xzf chkrootkit.tar.gz
cd chkrootkit-*
make sense
sudo cp chkrootkit /usr/local/bin/

# Monitoring
sudo apt install -y rsyslog logrotate

# Utilitaires
sudo apt install -y jq curl wget git

# Enable services
sudo systemctl enable ufw
sudo systemctl enable fail2ban
sudo systemctl enable clamav-daemon
sudo systemctl enable rsyslog

echo "✅ Installation complète"
echo ""
echo "Prochaines étapes :"
echo "1. Configurer UFW : sudo ufw enable"
echo "2. Configurer fail2ban : sudo systemctl start fail2ban"
echo "3. Update ClamAV : sudo freshclam"
echo "4. Scanner Lynis : sudo lynis audit system"
echo "5. Installer Vuls : voir guide dédié"
echo "6. Configurer LMD : maldet -u"
echo ""
echo "📚 Consulter les guides détaillés pour chaque outil"

Métriques de succès¶

À la fin du parcours, tu devrais avoir :

✅ Score Lynis ≥ 75/100 (hardening configuration)
✅ Vuls scan = 0 CVE critiques (9.0+ CVSS)
✅ fail2ban = 5+ jails actives (SSH, nginx, etc.)
✅ UFW = Politique deny by default + whitelist services
✅ ClamAV = Scan quotidien automatisé
✅ LMD = Monitoring temps réel /uploads (si serveur web)
✅ chkrootkit = Scan hebdo automatisé
✅ Logs = Rétention 90 jours + rotation
✅ Alertes = Email/Slack sur événements critiques
✅ Documentation = Runbook incident + checklist maintenance

Prochaines étapes¶

Une fois cette stack maîtrisée, tu pourras progresser vers :

  1. SIEM : Wazuh, ELK Stack (centralisation multi-hôtes)
  2. IDS/IPS réseau : Suricata, Snort (détection réseau)
  3. EDR : Osquery, Falco (monitoring forensique avancé)
  4. Compliance automation : OpenSCAP, InSpec
  5. Infrastructure as Code : Ansible/Terraform pour déploiement automatisé

La sécurité host est la fondation. Sans elle, tout le reste est fragile. 🛡️
